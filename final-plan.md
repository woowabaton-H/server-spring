# 청소 생활 앱 최종 기획서

작성일: 2026-07-10

## 1. 서비스 한 줄 정의

오늘 관리할 일은 놓치지 않고, 막히는 청소 문제는 함께 해결하는 청소 생활 앱

## 2. 배경 및 문제 정의

청소는 한 번의 큰 결심보다 작은 관리가 반복될 때 유지된다. 하지만 실제 생활에서는 언제 욕실을 마지막으로 청소했는지, 침구를 언제 세탁했는지, 쓰레기 배출을 며칠째 미뤘는지 기억하기 어렵다. 사용자는 청소가 필요하다는 사실은 알지만, 관리 주기를 스스로 설계하고 유지하는 데서 자주 막힌다.

막상 청소를 하려는 순간에도 문제는 이어진다. 검색 결과는 많지만 내 상황에 맞는 답을 고르기 어렵고, 상품 정보는 구매 중심으로 흘러가기 쉽다. 사용자는 지금 필요한 방법, 참고할 만한 용품, 맡길 수 있는 서비스, 다른 사람의 경험을 한곳에서 자연스럽게 확인하고 싶어 한다.

이 서비스는 청소를 완벽하게 관리하게 만드는 도구가 아니다. 사용자가 오늘 챙길 일을 놓치지 않고, 막히는 순간에 다음 선택지를 찾을 수 있게 돕는 생활 관리 앱이다.

## 3. 타깃 사용자

초기 타깃은 스스로 생활 공간을 관리해야 하지만 청소 기준과 리듬이 아직 안정되지 않은 개인 사용자다.

| 사용자 | 상황 | 필요한 도움 |
| --- | --- | --- |
| 자취 초기 사용자 | 집 관리 기준이 부족하다 | 기본적인 청소 주기와 시작점 |
| 바쁜 직장인 | 평일에 미루고 주말에 몰아서 처리한다 | 오늘 또는 이번 주에 챙길 일의 가벼운 정리 |
| 청소 초보 사용자 | 방법과 용품 선택이 어렵다 | 신뢰할 수 있는 팁과 선별된 추천 |
| 외부 서비스를 가끔 쓰는 사용자 | 맡길 수 있는 선택지를 매번 찾기 번거롭다 | 상황에 맞는 서비스 후보와 판단 기준 |

## 4. 사용자 페르소나

### 김보송, 26세, 자취 3년차 직장인

김보송은 퇴근 후 집안일을 자주 미룬다. 욕실, 세탁, 분리수거처럼 해야 할 일이 있다는 것은 알지만 언제 했는지 정확히 기억하지 못한다. 청소 앱을 써보려 했지만 세부 항목을 많이 입력해야 해서 오래 가지 않았다. 원하는 것은 완벽한 관리 시스템이 아니라, 이번 주에 무엇을 챙기면 되는지 알려주고 완료를 쉽게 기록하게 해주는 앱이다.

### 박루틴, 24세, 첫 자취 사용자

박루틴은 청소를 얼마나 자주 해야 하는지 기준이 없다. 청소 도구도 무엇부터 사야 할지 모르고, 검색 결과가 광고인지 실제 경험인지 구분하기 어렵다. 처음 쓰기 쉬운 방법과 용품, 다른 사용자의 질문 답변을 참고하면서 자기만의 관리 리듬을 만들고 싶어 한다.

| 가치 | 설명 |
| --- | --- |
| 기억 부담을 줄인다 | 사용자가 청소 주기를 모두 외우지 않아도 되게 한다 |
| 시작을 가볍게 만든다 | 세부 항목을 많이 등록하지 않아도 큰 흐름부터 관리할 수 있게 한다 |
| 막히는 순간을 줄인다 | 방법, 경험, 용품, 서비스 선택지를 한 흐름 안에서 찾게 한다 |
| 추천을 신뢰 가능하게 만든다 | 왜 추천하는지와 어떤 상황에 맞는지를 먼저 설명한다 |
| 다시 시작하기 쉽게 만든다 | 실패나 압박보다 다음 행동을 부드럽게 제안한다 |

## 6. 서비스 범위

MVP는 `주기 관리`, `문제 해결`, `선별 추천`, `기록`의 네 가지 축으로 구성한다.

| 영역 | 역할 |
| --- | --- |
| 주기 관리 | 사용자가 큰 청소 카테고리 단위로 관리 주기를 확인하고 완료한다 |
| 문제 해결 | 막히는 청소 문제를 커뮤니티의 꿀팁과 Q&A로 해결한다 |
| 선별 추천 | 브랜드 관점으로 고른 청소 용품과 서비스를 소개한다 |
| 기록 | 완료한 청소와 저장한 추천을 나중에 다시 확인한다 |

초기 제품은 넓은 기능보다 반복 사용을 만드는 데 집중한다. 사용자가 앱을 열고, 오늘 챙길 카테고리를 보고, 완료를 기록하고, 필요할 때 도움을 찾는 흐름이 자연스럽게 이어지는 것이 우선이다.

## 7. 주요 기능

### 7.1 청소 주기 관리

사용자는 욕실, 주방, 세탁/침구, 쓰레기/수거, 바닥/먼지, 계절/가전처럼 이해하기 쉬운 큰 카테고리로 청소 리듬을 관리한다. 각 카테고리는 마지막 완료일과 다음 관리 시점을 기준으로 보여주고, 사용자가 완료하면 다음 주기가 갱신된다.

### 7.2 브랜드 셀렉션

청소 용품과 서비스는 단순 목록이 아니라 선별된 추천으로 제공한다. 추천의 핵심은 “무엇을 팔 것인가”가 아니라 “어떤 상황에서 왜 도움이 되는가”다. 추천 항목에는 추천 이유, 적합한 상황, 가격대, 제휴 여부처럼 사용자의 판단에 필요한 정보를 함께 둔다.

### 7.3 커뮤니티

커뮤니티는 꿀팁 공유와 Q&A를 중심으로 운영한다. 사용자는 실제 경험에서 나온 관리 방법을 보고, 막히는 문제를 질문하거나 비슷한 답변을 찾아볼 수 있다. 반응이 좋은 글은 더 쉽게 발견되도록 정리한다.

### 7.4 마이 기록

마이 영역은 사용자의 관리 흔적을 보여준다. 완료한 카테고리, 최근 기록, 저장한 셀렉션을 모아 사용자가 자신의 청소 리듬을 확인할 수 있게 한다.

## 8. 핵심 사용자 시나리오

### 시나리오 A: 청소 주기를 관리한다

사용자가 앱을 열면 오늘 또는 이번 주에 챙기면 좋은 청소 카테고리를 확인한다. 사용자는 실제로 청소를 마친 카테고리를 완료 처리하고, 앱은 다음 관리 시점을 갱신한다. 주기가 생활 리듬과 맞지 않으면 사용자가 직접 조정한다.

이 시나리오는 MVP의 중심이다. 사용자가 큰 카테고리 단위의 주기 관리를 부담 없이 반복할 수 있는지가 제품의 첫 번째 검증 지점이다.

### 시나리오 B: 막히는 문제를 해결한다

사용자가 욕실 물때, 빨래 냄새, 분리수거 기준처럼 막히는 문제를 만난다. 커뮤니티에서 비슷한 꿀팁이나 Q&A를 확인하고, 필요하면 질문을 남긴다. 해결에 도움이 된 글은 저장해 다음에 다시 본다.

### 시나리오 C: 좋은 선택지를 참고한다

사용자가 청소 도구나 외부 서비스를 찾아야 할 때 셀렉션을 확인한다. 서비스는 많은 선택지를 나열하기보다 사용 상황에 맞는 후보를 골라 보여주고, 추천 이유와 판단 기준을 함께 제공한다. 사용자는 구매나 이용을 강요받지 않고 참고할 만한 선택지를 얻는다.

## 9. 전체 사용자 흐름

```text
앱 진입
→ 오늘/이번 주 관리할 청소 카테고리 확인
→ 완료 또는 주기 조정
→ 막히는 문제 발생 시 커뮤니티 또는 셀렉션 확인
→ 도움 되는 글이나 추천 저장
→ 완료 기록과 저장 항목 확인
→ 다음 관리 시점에 다시 방문
```

## 10. MVP 기준

MVP는 사용자가 청소 생활의 리듬을 만들 수 있는지 검증하는 단계다. 기능의 양보다 반복 사용 가능성이 중요하다.

| 기준 | 설명 |
| --- | --- |
| 기본 주기 제공 | 사용자가 처음부터 모든 기준을 만들지 않아도 된다 |
| 쉬운 완료 기록 | 청소를 마친 뒤 바로 기록할 수 있다 |
| 간단한 조정 | 사용자의 생활 패턴에 맞게 주기를 바꿀 수 있다 |
| 문제 해결 연결 | 막히는 순간 커뮤니티와 셀렉션으로 이어진다 |
| 저장과 회고 | 도움이 된 정보와 완료 기록을 다시 볼 수 있다 |

## 11. 추후 확장 방향

초기 사용성이 검증되면 다음 방향을 단계적으로 검토한다.

- 생활 패턴에 따른 주기 추천 고도화.
- 계절, 날씨, 공간 특성에 따른 관리 제안.
- 청소 용품과 서비스 셀렉션의 카테고리 확장.
- 커뮤니티 답변 품질 관리와 전문가 검수.
- 외부 서비스 연결 경험의 정교화.

## 12. 운영 고려사항

### 12.1 셀렉션 운영

셀렉션은 신뢰가 핵심이다. 추천은 판매 문구보다 사용 맥락과 판단 근거를 먼저 설명해야 한다. 제휴 여부는 숨기지 않고, 가격은 변동 가능성을 전제로 다룬다. 사용자가 직접 해결할 수 있는 방법이 있다면 함께 안내한다.

### 12.2 커뮤니티 운영

커뮤니티는 실제 경험과 질문 답변이 쌓이는 공간이어야 한다. 광고성 글, 위험한 청소 방법, 검증되지 않은 과장 정보는 운영 기준에 따라 관리한다. 좋은 글은 더 쉽게 발견되게 하되, 특정 상품이나 서비스로 과하게 유도하지 않는다.

### 12.3 알림과 표현

알림은 압박보다 기억 보조에 가깝게 설계한다. 사용자가 못한 일을 지적하기보다 이번 주에 챙기면 좋은 일을 알려주는 톤이 적합하다.

## 13. 청소 도메인 고려사항

- 청소 주기는 생활 패턴, 계절, 공간 크기, 사용 빈도에 따라 달라진다.
- 세제나 도구 추천에는 소재 손상, 환기, 혼합 금지 같은 안전 정보가 필요하다.
- 외부 서비스는 가격, 제공 범위, 지역, 일정, 후기의 변동성이 크다.
- 사용자는 완벽한 청소보다 다시 시작할 수 있는 기준을 더 필요로 한다.
- 청소 정보는 경험 기반 팁과 안전 기준을 함께 다뤄야 신뢰가 생긴다.

## 14. 차별화 포인트

1. 세부 작업 목록보다 큰 청소 리듬을 먼저 잡는다.
2. 사용자가 오늘 챙길 일을 놓치지 않게 돕는다.
3. 막히는 문제를 커뮤니티와 선별 추천으로 자연스럽게 연결한다.
4. 상품과 서비스는 판매 목록이 아니라 브랜드가 고른 선택지로 제안한다.
5. 압박보다 다시 관리할 수 있는 감각을 만든다.

## 15. 성공 지표

| 단계 | 지표 | 의미 |
| --- | --- | --- |
| 활성화 | 첫 카테고리 완료율 | 사용자가 핵심 행동을 이해했는가 |
| 반복 | 주간 완료 사용자 비율 | 주기 관리가 반복 사용으로 이어지는가 |
| 조정 | 주기 수정률 | 사용자가 자기 생활에 맞게 앱을 조정하는가 |
| 문제 해결 | 커뮤니티 글 조회와 저장 | 막히는 문제 해결에 도움이 되는가 |
| 셀렉션 | 셀렉션 저장과 외부 이동 | 선별 추천이 참고 가치가 있는가 |
| 리텐션 | D7/D30 재방문율 | 청소 주기가 재방문 이유가 되는가 |

North Star Metric은 `월간 청소 카테고리 완료 기록 수`로 둔다. 사용자가 청소 생활을 실제로 관리하고 있는지 가장 직접적으로 보여주는 지표다.
